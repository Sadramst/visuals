# Appilico Intelligence Suite — Architecture

## Overview
9-dashboard operational intelligence platform for Australian mining executives.
Built as a Turborepo monorepo with Power BI custom visuals (D3.js) and React wrapper components.

## Visual Package Structure
Each visual has a dual architecture:

### `pbiviz/` — Power BI Custom Visual
- **API**: powerbi-visuals-api ~5.8.0
- **Rendering**: D3.js ^7.8.0
- **Language**: TypeScript 5.4 (`module: "node16"`, `moduleResolution: "node16"`)
- **Entry**: `src/visual.ts` implementing `IVisual`
- **Pattern**: Constructor creates SVG/DOM → `update()` parses DataView → renders

### `react/` — React Wrapper Component
- **Framework**: React 18 with Recharts ^2.12.0
- **Animations**: Framer Motion ^11.0.0
- **Build**: TypeScript with JSX (`module: "ESNext"`, `moduleResolution: "bundler"`)
- **Pattern**: Functional component with `useMemo` for data transformation

## Data Model — Star Schema
15 tables: 7 dimensions + 8 fact tables.
Generated by `data-generation/generate.py` producing 24 months × 5 sites.

### Dimensions
| Table | Rows | Key Fields |
|-------|------|-----------|
| dim_date | 721 | date_key, fiscal_year, is_shutdown |
| dim_equipment | 19 | equipment_id, type, capacity_tonnes |
| dim_crew | 4 | crew_id, rotation_type |
| dim_location | 5 | location_id, state, latitude/longitude |
| dim_material | 5 | material_id, material_type |
| dim_cost_category | 9 | category_id, is_controllable |
| dim_kpi | 12 | kpi_id, domain, unit |

### Facts
| Table | Rows | Grain |
|-------|------|-------|
| fact_production | ~7.2k | site × date |
| fact_equipment_status | ~127k | equipment × hour |
| fact_ore_grade | ~805 | site × month |
| fact_cost | ~1k | site × category × month |
| fact_safety | ~1.4k | site × KPI × month |
| fact_emissions | ~805 | site × source × month |
| fact_roster | ~15.6k | crew × site × date |
| fact_haulage | ~14.1k | route × site × date |

## CI/CD
GitHub Actions: 3 jobs (Node.js Build & Test, .NET Build, Package PBI Visuals).
Turborepo orchestrates `build`, `test`, `lint` across all packages.

## Key Conventions
1. `.js` extensions on all local imports (required by `moduleResolution: "node16"`)
2. D3 data bindings: cast with `as TypeName` (D3 v7 infers `unknown`)
3. `strict: false` in pbiviz tsconfigs (Power BI API compatibility)
4. Package-level scripts: `build: "cd react && tsc"`, `test: "exit 0"`, `lint: "exit 0"`
